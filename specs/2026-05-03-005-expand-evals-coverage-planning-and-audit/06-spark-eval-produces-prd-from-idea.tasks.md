## Slice 1: Author the spark-from-idea YAML scenario

**Goal**: `evals/cases/spark-from-idea.yaml` exists, is auto-discovered by `loadScenarios`, and `npm run eval -- --case spark-from-idea` reports PASS against a one-line free-text idea prompt — regardless of whether smithy-survey returns a populated `### Alternatives Considered` table or the documented empty-state stub — with sub-agent evidence covering smithy-survey, smithy-clarify, and smithy-prose.

**Justification**: Spark is the only US6-class scenario that requires no planted fixture artifact — its prompt is the idea text itself, and the data-model + contracts §4 explicitly forbid an `evals/fixture/prds/spark-eval/` directory. That collapses the sibling US3/US4/US5 plant-then-YAML two-slice pattern to a single deliverable: the YAML scenario. Spark also issues no `git checkout -b`, `git commit`, or `gh pr create` (verified empty against the spark template), so this story has **no hard cross-story dependency** on US2 Slice 1's runner git-init — the slice is independently mergeable in any order relative to all siblings. Empirical confirmation of `required_headings` and the smithy-prose marker is folded into the authoring task's acceptance criteria rather than split into a separate investigation task; sibling US5 Slice 2 sets the single-task precedent and the cut prompt's "no research/file-reading tasks" rule forbids a standalone investigation step. The reconciler also surfaced and rejected a second README-row task — sibling READMEs document **actual** plants and their realism rating, so a `(none — free-text prompt)` row would document non-existence and miscategorize the registry; the data-model's explicit prohibition is the canonical documentary signal.

**Addresses**: FR-001, FR-002, FR-003, FR-008 (survey alternation), FR-009, FR-011, FR-012, FR-013, FR-014; AS 6.1 (terminal PRD-path marker + extracted-text structural anchor), AS 6.2 (populated `### Alternatives Considered` alternation branch), AS 6.3 (empty-state stub alternation branch); AS 6.4 carried as story-level debt (see SD-013); SC-001, SC-002, SC-003, SC-005, SC-006.

  - `required_patterns` contains a regex alternation matching either `### Alternatives Considered` (the populated-table heading from smithy-survey's success path — satisfies AS 6.2) OR `No comparable off-the-shelf options identified during survey\.` (the empty-state stub's opening sentence from `src/templates/agent-skills/agents/smithy.survey.prompt` — satisfies AS 6.3); the alternation satisfies FR-008 in a single pattern entry; an inline YAML comment cross-references the survey template's empty-state fallback block and inherited SD-005
  - `sub_agent_evidence` has exactly three entries covering smithy-survey, smithy-clarify, and smithy-prose; **smithy-plan-review is deliberately omitted** because the spark template dispatches zero plan-review agents (verified empty); an inline YAML comment cross-references SD-016 and the spec matrix discrepancy
  - The smithy-survey entry uses `^[Ss]urvey` (dispatch-description marker, strike-precedent shape from contracts §3)
  - The smithy-clarify entry uses `^[Cc]larif` (strike-precedent marker from `evals/cases/strike-health-check.yaml`)
  - The smithy-prose entry uses an empirically captured template-stable marker (candidates: dispatch-description marker `^[Pp]rose`, or the leading content of the prose return — final choice pinned in an inline YAML comment per the sibling-precedent mitigation pattern, resolves SD-015)
  - `forbidden_patterns` contains at minimum `"I'd be happy to help"`, `"Sure, here's"`, and `'^---\r?\n'` (FR-011)
  - `timeout:` field is omitted unless empirical calibration shows spark exceeds 120s; if present, an inline comment records the observed wall-clock (per inherited SD-006, expected unused for spark)
  - YAML loads without warning via `loadScenarios`; `npm run eval -- --case spark-from-idea` selects only this scenario and reports PASS regardless of which survey branch fires (SC-001, SC-002)
  - Strike, scout, and any landed sibling scenarios continue to PASS unchanged after this scenario is added (FR-012, SC-003)
  - `npm run test:evals` (offline unit tests) continues to pass with the new YAML file present (SC-006)
  - No changes to `evals/lib/runner.ts`, `evals/lib/scenario-loader.ts`, `evals/lib/structural.ts`, or any other framework module (FR-002, FR-003)

**PR Outcome**: `npm run eval -- --case spark-from-idea` exercises `/smithy.spark` with a free-text idea prompt and reports PASS whether smithy-survey returns a populated alternatives table (AS 6.2) or the empty-state stub (AS 6.3). Sub-agent evidence covers the three agents spark's template actually dispatches (survey, clarify, prose). AS 6.4's unimplementability is documented as SD-013 and not asserted in the scenario. The scenario requires no fixture plant, no git-init dependency, and no `gh` auth — independently mergeable. Inherited debt SD-005 transitions toward `resolved` for the spark variant once the YAML scenario lands with the survey-template cross-reference comment.
