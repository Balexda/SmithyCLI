## Summary

Add a `smithy doctor` CLI command (alias `health`) that verifies the integrity of a smithy installation. It reads the manifest and checks that everything is in order ŌĆö no file modifications, purely diagnostic.

## Approach

**New file: `src/commands/doctor.ts`** ŌĆö single self-contained command handler following the existing pattern of `init.ts`, `uninit.ts`, `update.ts`.

The command performs these checks:

1. **Manifest exists** ŌĆö checks both `repo` and `user` locations. Distinguishes "missing" from "corrupt" by pre-checking `fs.existsSync` on the manifest path before calling `readManifest()` (which silently swallows parse errors). `[via Robustness]`
2. **Deployed files present** ŌĆö iterates `allManifestFiles(manifest)`, verifies each exists on disk.
3. **Version current** ŌĆö compares `manifest.smithyVersion` against the running CLI's `smithyVersion`.
4. **Permissions file exists** ŌĆö if `manifest.permissions === true`, verifies the permissions file is present. `[via Robustness]`

**Output**: Colored terminal summary via `picocolors` (green=OK, red=error, yellow=warn). Exit code 1 if any errors, 0 otherwise.

**CLI registration in `src/cli.ts`**:
```
program.command('doctor').alias('health')
  .option('-d, --target-dir <path>', 'Target directory')
  .action(doctorAction)
```

**Tests in `src/commands/doctor.test.ts`** ŌĆö unit tests with temp directories covering: no manifest, healthy install, missing files, version mismatch, corrupt manifest.

## Risks

| Risk | Mitigation |
|------|------------|
| `readManifest` can't distinguish missing from corrupt | Pre-check with `fs.existsSync` before calling `readManifest` |
| `compareSemver` is private in `update.ts` | Duplicate a simple version comparison inline (2 call sites doesn't warrant extraction yet) |
| Version comparison doesn't handle pre-release tags | Acceptable ŌĆö `smithyVersion` uses plain semver |

3. **Deferred features**: `--json` output, `--fix` flag, content hashing, and drift detection all deferred to future iterations.
